# 托马斯数字艺术学园宪章

**The Collage Garden · 拼贴花园**

---

## 序言

我上二年级时，启蒙老师的姐姐正在学用3ds建模一个茶壶。那是我与数字艺术的初遇。此后三十年，我一直在追逐技术——操作系统、网络、数据库、机器学习、自然语言处理、物联网、计算机视觉、大型语言模型——一直在跑，离那个茶壶越来越远。

如今四十岁了，我想走回去。

不是以专家的身份回去教别人，而是以一个从未真正开始的学习者身份。这座学园是一个花园，我在这里种下我所学到的东西，任何人都可以走进来，捡起什么，或者种下自己的东西。

这份宪章就是这座花园的宪法。

---

## 第零条——灵魂

**我们的教育不是教会学生做对什么，而是教会他们如何最低成本地犯错，如何从不断的犯错中学习并改善自己的作品。**

一件展出的作品不是完成了——它是在*进行*。如果我们永远不让学习者尝试他们不确定的事情，他们怎么能体验到未知的力量？

本条优先于所有其他条款。

---

## 八条原则

### 一、没有"完成"，只有版本

这座花园里的每一份内容都有版本。初稿可以粗糙，甚至可以是错的。犯错不丢人——我们的核心承诺不是正确，而是诚实的记录。Git历史本身就是一本教科书：任何人都可以追溯一个想法如何从模糊走向清晰。

### 二、从真实问题出发，永远不从定义开始

任何内容都不应以"让我们来学习傅里叶变换"开头。它必须以"为什么二胡的声音和小提琴听起来不一样？"开头——然后在追寻答案的过程中，傅里叶变换自然而然地出现了。场景优先不仅是组织方法，更是一种创作纪律。

### 三、标注来源，拒绝装懂

我们将广泛学习和引用他人的作品。所有引用必须正确标注出处。当有什么不理解的地方，就写"我还没理解这部分"。这不是软弱——这是我们的品牌。诚实比博学更重要。

### 四、作品优先于笔记

这座花园的最终产出不是读书笔记——而是作品。一段音频、一张生成的图片、一个交互网页、一个3D模型（是的，终于要做那个茶壶了）。每段学习旅程都应该以"这是我们用所学做出的东西"结尾。笔记记录过程；作品标记里程碑。

### 五、六条场景，交叉的路径

六条场景——**日常生活**、**社会观察**、**听觉艺术**、**视觉艺术**、**互动装置**、**传统艺术**——不是平行的轨道。它们是交叉的花园小径。一个关于城市噪音的项目可以同时属于听觉艺术和社会观察。一个互动书法装置生长在互动装置与传统艺术的十字路口。内容结构必须支持跨场景的交叉引用。没有任何知识被锁在一个抽屉里。

### 六、一个好奇的大一新生就能参与

我们的学习者从艺术学生到研究生研究者不等。入门门槛必须设定在一个好奇的、没有编程基础的大一新生能够进入的水平。深度可以无限延伸，但门必须低到任何人都能走进来。每个话题都从假设读者什么都不知道开始。

### 七、AI协作，透明标注

这座学园从第一天起就与AI共同创作。所有AI贡献的内容都应透明标注——不是为了划界线，而是因为诚实是我们的第一原则。读者有权知道。这也是对教育未来的一次示范：人类与AI的协作不是作弊，但它需要透明。

### 八、没有截止日期——花园按自己的节奏生长

这个项目不应成为压力的来源。没有截止日期、没有更新承诺。花园按维护者的节奏生长。有的季节繁茂，有的季节休眠。两者都是自然的。

## 基础设定

### 语言

主要语言：**英语**。当某个话题达到相对完整的状态时（按原则一，这永远是暂时的），将创建多语言版本。

### 许可协议

- **内容**（文字、图片、音频、课程）：[知识共享 署名-相同方式共享 4.0 国际 (CC BY-SA 4.0)](https://creativecommons.org/licenses/by-sa/4.0/)
- **代码**（脚本、工具、配置）：[MIT 许可协议](https://opensource.org/licenses/MIT)

### 内容组织

内容按**场景**（六条花园小径）组织，由来自不同学科的**知识碎片**提供支持。

**场景**——穿过花园的六条路径：

| 场景 | 它探索什么 |
|---|---|
| **日常生活** | 隐藏在日常中的艺术与科学 |
| **社会观察** | 社会如何运转、人如何互动、文化如何演变 |
| **听觉艺术** | 声音、音乐、声学，以及聆听的体验 |
| **视觉艺术** | 图像、形态、色彩、空间，以及观看的体验 |
| **互动装置** | 技术与物理空间、人的参与相遇的地方 |
| **传统艺术** | 以现代视角重新审视古老技艺 |

**知识碎片**——工具架，不是课表：

| 学科 | 它提供什么 |
|---|---|
| **物理学** | 力学、声学、光学、热力学、电磁学 |
| **数学** | 几何、代数、微积分、统计学、信号、信息论 |
| **经济学** | 市场、行为经济学、博弈论、资源配置 |
| **哲学** | 美学、认识论、伦理学、逻辑学、现象学 |
| **心理学** | 感知、认知、情感、创造力、行为模式 |
| **生理学** | 感觉系统、运动控制、具身体验 |
| **神经科学** | 神经处理、听觉/视觉皮层、可塑性、意识 |

这些知识碎片被系统化地组织起来以便在需要时被找到——但它们存在是为了**服务场景**，而不是主导它们。学生不是来这座花园"学物理"的。学生是来做东西的，在过程中发现自己需要理解振动，在知识架上找到所需的内容，然后回到自己的工作中去。

### 内容形式

- **故事** — 创作过程本身：创作日记、学习叙事、走进的死胡同、灵感的火花、意想不到的转折。不是每个故事都会变成作品，但每件作品都从一个故事开始。故事是根；作品是果。
- **知识笔记** — 学习碎片、参考资料、探索
- **场景作品** — 基于场景学习诞生的创作
- **课程** — 由故事、笔记和作品组装而成的结构化学习路径

### 角色

- **维护者**：Thomas（唐云峰）— 学习者、园丁、宪章守护者
- **协作者**：AI伙伴 — 共同创作内容、挑战想法、提供视角
- **贡献者**：所有人 — 欢迎共同创作、分享作品、漫步花园小径
