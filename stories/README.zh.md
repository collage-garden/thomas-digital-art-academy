# 故事

*果实底下的根系。*

不是每段旅程最后都能拿出一件作品——但每段旅程都是故事，值得被讲出来。灵感怎么来的，死胡同怎么撞的，拐弯处遇到了什么意外，某个瞬间突然想通了——这些就是故事。

没有故事的作品是空的。没有变成作品的故事照样完整——因为做的过程本身，和做出来的东西一样重要。

---

## 故事长什么样

- **创作日记**——做一样东西的全过程，一步一步，包括翻车
- **学习叙事**——追一个问题的路径，走错的弯路也留着
- **协作记录**——人和人（以及人和 AI）怎么围绕一个念头合作
- **碎片**——一颗火花，还没找到形状，但值得记住

故事和[场景](../scenes/)、[知识](../knowledge/)、[作品](../works/)自然相连。故事可能长成作品，作品应该永远指回它的故事。
